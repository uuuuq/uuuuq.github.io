FY3C/MERSI band5反演星上亮温

发表于 2015-04-10   |   分类于 学习   |  

FY3C/MERSI band5是热红外波段，可反演地表温度。总之又是缺少文档，经过与气象局工作人员的沟通才最终搞定。

band5存储在L1数据的EV_250_ Emissive和EV_250_Aggr.1KM_Emissive两个科学数据集中，内容为红外辐亮度，存储系数为0.01，单位为$mW\cdot m^{-2}\cdot cm^{-1}\cdot sr$。EV_250_ Emissive的分辨率为250M，EV_250_Aggr.1KM_Emissive
为重采样至1KM分辨率的红外辐亮度。

中心波数$w_n=913.7016cm^{-1}$，根据普朗克公式计算得到等效黑体亮温$T_e$：$T_e=Plank^{-1}(radiance,wn)$，再将$T{e}$转换为黑体亮温$T{bb}$：$T{bb} = A*T_e+B$，其中$A=1.0047$，$B=-0.8549$。

得到亮温后可用单通道算法反演地表温度。

FY-3C MERSI数据几何校正

发表于 2015-03-28   |   分类于 学习   |  

更新一则讣告：
FY-3C MERSI
2013.09.30-2015.05.31

最近用了FY3C/MERSI数据，总体上来说比A星和B星都要好很多。AB上的热红外波段(band5)传感器有问题，是利用VIRR对应波段数据替换的，目前C星上该波段工作状态良好；后续波段存在的stripe noises问题也得到了有效的抑制。数据的组织结构也进行了一些调整。

FY3C和之前的两颗星一样，只提供L1数据，拿到手上后需要自己进行定标和校正。

FY3C/MERSI L1数据结构

L1数据包括四种文件:

中分辨率光谱成像仪L1数据(1KM_GEO)
中分辨率光谱成像仪L1数据(250M_GEO)
中分辨率光谱成像仪L1数据(1KM)
中分辨率光谱成像仪L1数据(250M)

四种文件均以HDF5格式存储。1和2内存放的是Geolocation信息，3和4内保存的是反射/发射数据。MERSI的前5个波段为250m分辨率，保存在250M文件中；后15个波段为1000m分辨率，保存在1KM文件中，另外该文件中还包含了重采样为1KM的b1-b5波段数据。

1KM 文件包含3个Group，每个Group下又包含若干个Dataset，其组织结构如下(250KM与其类似)：

Calibration下的IR_Cal_Coeff和VIS_Cal_Coef存储的是定标系数；Data下EV_1KM_RefSB，EV_250_Aggr.1KM_Emissive， EV_250_Aggr.1KM_RefSB存储的分别是b6-b20、重采样为1KM的b5以及重采样为1KM的b1-b4，其他Dataset对于普通用户意义不大。QA_index中存储的是Quality Assurance，也是很重要的数据，不过没查找的相关的说明文档，不知道该怎么用。

几何校正

利用ENVI/IDL进行几何校正

ENVI5.0后自带HDF5文件读取模块，如果使用的是ENVI Classic或是4.8及之前的版本，需要手动加入一个读取HDF5数据的扩展工具，此处以ENVI Classic + HDF5数据扩展工具为例。

以ENVI + IDL模式运行程序，利用HDF5扩展工具选中需要校正的Dataset，import to ENVI。正常情况下会弹出Available Band List，包含了刚刚打开的Dataset，记住勾选include data，否则导入的是结构体，需要在idl内处理后用import from IDL variable导入。

导入待校正的Dataset和对应分辨率的Latitude及Longitude，利用Georeference from input Geometry - Build GLT生成GLT文件，再用Georeference from GLT功能即可进行校正。具体方法网上有，不详述。需要注意的是，FY3A和FY3B都只有1KM的Latitude和Longitude，如果要校正250M的反射率，需要先将1KM的经纬度数据重采样成250M，再进行校正。由于数据质量问题，有时会数据中出现坏扫描线，在Latitude和Longitude中以-999进行填充，如果直接build GLT会出错，比较坏的情况是会生成一个巨大无比的GLT文件，直到把整个硬盘全部填满然后报错。

用批处理模式调用ENVI API可以方便地对大量数据进行校正。大致思路就是先读取Latitude/Longitude和待校正的数据，再调用envi_glt_doit和envi_georef_from_glt_doit进行校正。需要定义输出图像的projection信息，这里写死了。
